03Definitions

What is generative AI?

AI that doesn't just classify the world — it produces new artifacts inside it. Text, code, images, audio, video.

01 · 1950s — 2000s

Rule-based

Humans wrote the rules. The machine followed them. Brittle, narrow, but explainable.

→
02 · 2000s — 2017

Learning-based

Models learned patterns from labeled data — classify, predict, rank. Powerful, but discriminative.

→
03 · 2017 — now

Generative

Models learn the distribution itself — sample from it to create. The interface becomes language.

04Mechanics

How LLMs work, simply

An LLM is a machine that predicts the next token — a fragment of text — given everything before it. Repeat the prediction, and you get a paragraph. Scale the parameters, and the predictions become strikingly fluent.

CONCEPT · 01

Tokens

Sub-word chunks the model actually sees. "Tokenization" ≠ words.

CONCEPT · 02

Context window

How much text it can hold at once. The model's short-term memory.

CONCEPT · 03

Parameters

The learned weights — billions of them. Scale is most of the magic.

06Practice

Prompting is engineering

A prompt is a specification. The clearer the spec, the more reliable the system. Three patterns cover most of the ground.

PATTERN · 01

Zero-shot

Just ask. No examples. Fine for well-known tasks the model has seen a million times.

"Summarize this email in one sentence."
PATTERN · 02

Few-shot

Show the model two or three examples of the input → output shape. It will copy the pattern.

"Input: cat → Output: feline
Input: dog → Output: canine
Input: horse → Output:"
PATTERN · 03

Chain‑of‑thought

Ask the model to think step by step. Reasoning quality jumps; especially on math, logic, and code.

"Let's solve this step by step. First, …"

Common beginner mistakes — vague verbs, missing constraints, no examples, and asking for "creative" output without a frame.

07The Failure Mode

An LLM does not know what is true. It knows what is likely.

That gap — between likely and true — is where hallucinations live. The model invents a plausible answer because plausibility is what it was trained to produce. The fix is not "tell it not to" — the fix is architectural: ground the model in real sources (next slide), constrain outputs, and verify before you trust.

08Architecture

Retrieval‑Augmented Generation

Give the model an open book. Before it answers, fetch the relevant pages — then let it read and write.

STEP · 01

Retrieve

Search your own data (docs, tickets, code) for chunks relevant to the user's question.

→
STEP · 02

Augment

Stuff those chunks into the prompt as context. "Here are the facts. Answer using only these."

→
STEP · 03

Generate

The model writes the answer, grounded in retrieved evidence — and cites it.

Closed-book exam

Answer from memory alone. Confident — often wrong. Frozen at training time.

Open-book exam

Look it up first, then write. Slower, but defensible — and always up to date.

Embeddings & vector databases

An embedding turns a piece of text into a list of numbers — a coordinate in a high-dimensional space. Texts that mean similar things land near each other. Cosine distance between vectors becomes semantic similarity.

A vector database stores millions of these embeddings and finds nearest neighbors in milliseconds. It is the backbone of every RAG system.

10Architecture

Agents & workflows

A single prompt is one move. An agent is a game: the model plans, calls tools, reads results, and decides what to do next.

The loop

  • →Plan — break the goal into sub-tasks
  • →Act — call a tool (search, code, DB, API)
  • →Observe — read the result back into context
  • →Decide — try the next step, or stop

What you'll actually use

  • 01Tool / function calling — typed actions the model can invoke
  • 02Structured outputs — JSON schemas, not free text
  • 03LangGraph / state machines — explicit nodes & edgesWhen chains aren't enough, graphs are.

11Practice

Building in production

The part they don't teach you: making a demo that wins a hackathon is one thing. Making it not fall over at 3 a.m. on a Tuesday is another.

CONCERN · 01

Clean architecture

Service layers, dependency injection, typed interfaces. FastAPI + Pydantic is a great starting stack. Don't put your prompts in your route handlers.

CONCERN · 02

Reliability & cost

Models fail. Retries, timeouts, fallbacks, caching. Track tokens per request like you'd track DB queries — they're your new bill.

CONCERN · 03

Evaluation

You can't ship what you can't measure. Build a small eval set early. Diff outputs across model versions. Prompts are code.

A boring backend with a great LLM  >  a great LLM with a chaotic backend.

12Optimization

Fine-tuning & model optimization

When NOT to fine-tune

You haven't tried better prompts yet.

You don't have a clean dataset.

The base model already does the task ≥ 90% well.

You need facts to stay current — that's a RAG problem.

When fine-tuning earns its keep

You need a consistent style or output format.

You're running a narrow, repetitive domain task.

You want a small model to do what a big model does — cheaper, faster, on your own hardware.

13The Market

Career paths in GenAI

Four flavors. Most people drift between them — but it helps to know what each one actually looks like.

Role · 01

AI / ML Engineer

Ships ML systems end-to-end. Owns pipelines, training jobs, and the infrastructure underneath. The deepest path.

Role · 02

Backend AI Developer

Builds the APIs, RAG pipelines, and agent loops on top of foundation models. The fastest-growing path right now.

Role · 03

Prompt Engineer

Specializes in eliciting reliable behavior from models — evals, prompt patterns, guardrails. Often hybrid with a product or QA role.

Role · 04

AI Researcher

Pushes the frontier itself — new architectures, training methods, alignment. Usually a PhD path, or strong publications.

Skills that matter most: Python, system design for AI, API integration, and a portfolio of real things you've shipped.

What separates a junior from a senior in this space isn't more frameworks. It's system design thinking — knowing when a problem is a prompt, a RAG, a fine-tune, or a plain old SQL query.

"Engineering discipline + AI knowledge" is the rare combination companies pay for.

15The Market

Freelance & startup opportunity

Most local businesses know they should be using AI. Almost none of them know how. That gap is the opportunity.

Where the work is

  • Workflow automations & internal copilots
  • Document processing (invoices, contracts, intake forms)
  • Customer-support bots grounded in their docs
  • Lead-gen and content pipelines
  • Bespoke RAG on a company's own knowledge base

Why now

Big-tech consultants are too expensive. The competition at the local and regional level is still thin. A working prototype solves the trust problem faster than a deck.

Charge for the outcome, not the hours.

16The Path

A practical 4-month roadmap

MONTH · 01

Foundations

  • Python — modules, types, async
  • The OpenAI / Anthropic API
  • Prompting patterns: zero-shot, few-shot, CoT
  • One small project — a CLI that does something useful
MONTH · 02

Retrieval

  • Chunking & embedding strategies
  • pgvector or Chroma
  • Build a RAG app on your own notes / a public dataset
  • Measure: groundedness, faithfulness
MONTH · 03

Agents

  • Function / tool calling
  • Structured outputs, JSON schemas
  • LangGraph (or your own state machine)
  • Add a real tool: search, code-runner, DB
MONTH · 04+

Ship it

  • Fine-tune a small open model with LoRA
  • Deploy on a cheap GPU / serverless
  • Write it up — blog, GitHub, LinkedIn
  • Charge someone money for it
